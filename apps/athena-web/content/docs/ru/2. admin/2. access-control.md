# Система контроля доступа (ACL)

Архитектура безопасности Athena LMS построена на гибридной модели, сочетающей **RBAC** (Role-Based Access Control) и элементы **ABAC** (Attribute-Based Access Control).

## Основные понятия

### 1. Permission (Разрешение)
Это атомарное право на выполнение определенного действия. Разрешения представляют собой строки вида `ресурс.действие`.
* **Пример:** `courses.create` — право создавать курсы.

### 2. Role (Роль)
Именованный контейнер, содержащий список разрешений. Пользователь всегда имеет ровно одну роль.

### 3. Policy (Политика)
Дополнительное условие, ограничивающее действие разрешения. Политики позволяют реализовать логику "Пользователь имеет право редактировать курс, **НО ТОЛЬКО** если он является его автором".

Политики применяются поверх разрешений. Если разрешение отсутствует, политика не имеет смысла.

## Справочник политик

На данный момент система поддерживает следующие политики объектов:

| Политика | Описание | Применяется к |
| :--- | :--- | :--- |
| `OWN_ONLY` | Разрешает действие только над объектами, где `ownerId` совпадает с ID пользователя. | Курсы, Файлы |
| `NOT_PUBLISHED` | Разрешает действие только если объект находится в статусе черновика. | Курсы |
| `ONLY_PUBLISHED` | Разрешает доступ только к опубликованным объектам. | Курсы, Уроки |
| `PUBLISHED_OR_OWNER` | Комбинированное правило: видит опубликованные (как студент) И свои собственные (как автор). | Списки курсов |

## Матрица разрешений

Ниже приведен полный список доступных в системе прав, сгруппированных по модулям.

### Аккаунты 
Управление доступом в систему.
* `accounts.create` — Создание пользователей.
* `accounts.read` — Просмотр списка пользователей.
* `accounts.update` — Редактирование (смена ролей, сброс пароля).
* `accounts.delete` — Архивация/удаление.

### Профили
Личные данные пользователей.
* `profiles.create` — Создание профиля.
* `profiles.read` — Просмотр анкетных данных.
* `profiles.update` — Изменение данных.
* `profiles.delete` — Удаление профиля.

### Курсы 
Образовательный контент.
* `courses.create` — Создание нового курса.
* `courses.read` — Просмотр курсов.
* `courses.update` — Редактирование настроек курса.
* `courses.delete` — Удаление курса.
* `courses.publish` — Публикация курса (доступ для студентов).

### Уроки 
Контент внутри курса.
* `lessons.create` — Добавление материалов.
* `lessons.read` — Просмотр материалов.
* `lessons.update` — Редактирование контента.
* `lessons.delete` — Удаление.

### Прогресс и Зачисления
* `enrollments.create` — Зачисление студентов на курс.
* `enrollments.read` — Просмотр списков студентов.
* `progress.read` — Просмотр успеваемости.
* `progress.update` — Выставление оценок.

### Система
* `admin` — Глобальный доступ (Супер-Админ). Отключает проверки политик.
* `blocks.execute` — Право на запуск изолированного кода (Code Runner).
* `schedule.create` / `read` / `update` — Управление расписанием.
