# Управление Ролями

Ролевая модель в Athena LMS позволяет гибко настраивать доступ пользователей к функционалу системы. Вместо назначения прав каждому отдельному пользователю, администратор создает **Роль** (например, "Методист"), наделяет её набором разрешений и назначает эту роль пользователям.

## Список ролей

В разделе **Роли** отображается список всех доступных в системе конфигураций доступа.

В таблице представлена следующая информация:
* **Название:** Уникальное имя роли.
* **Уровень доступа:** Количество выданных прав или специальный бейдж **Супер-Админ**, если роль обладает полными правами.
* **Дата создания/обновления.**

::note{icon="i-lucide-info"}
Поиск по таблице осуществляется по названию роли.
::

## Создание и Редактирование

Для создания новой роли нажмите кнопку **Создать роль**. Откроется панель конфигуратора.

::steps{level="4"}
#### Название роли
Введите понятное название (минимум 3 символа). Оно должно быть уникальным в рамках системы.

#### Настройка прав
В интерфейсе права сгруппированы по функциональным модулям (Система, Курсы, Пользователи и т.д.).
Отметьте галочками действия, которые будут разрешены пользователям с этой ролью.

#### Сохранение
Нажмите **Сохранить**. Изменения вступают в силу немедленно, однако активным пользователям может потребоваться перезаход в систему для обновления токена доступа.
::

### Супер-Администратор

В системе существует особое разрешение — `admin` (Супер-Админ).
Если выдать роли это право, система будет **игнорировать** любые другие ограничения для этой роли. Пользователь получит полный доступ ко всем функциям и данным.

::warning{icon="i-lucide-triangle-alert"}
Будьте крайне осторожны при создании ролей с флагом Супер-Админа.
::

## Удаление ролей

Вы можете удалить роль, нажав на иконку корзины.

**Ограничение целостности данных:**
Система не позволит удалить роль, если она назначена хотя бы одному пользователю. Это защита от появления "потерянных" аккаунтов без прав доступа.

Если при удалении возникает ошибка, сначала переназначьте пользователей на другую роль в разделе **Пользователи**.
