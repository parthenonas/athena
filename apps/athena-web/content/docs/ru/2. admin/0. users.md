# Управление пользователями

Раздел **Пользователи** предназначен для администрирования учетных записей всех участников системы: студентов, преподавателей и других администраторов. Здесь вы можете создавать новые аккаунты, назначать роли, сбрасывать пароли и управлять статусом доступа.

## Просмотр списка

При входе в раздел отображается таблица всех пользователей системы.

::card-group
  ::card{title="Поиск" icon="i-lucide-search"}
  Поле поиска позволяет фильтровать пользователей по **Логину**. Поиск работает по частичному совпадению.
  ::
  ::card{title="Фильтрация" icon="i-lucide-filter"}
  Список можно сортировать по дате регистрации, логину или статусу.
  ::
::

## Создание учетной записи

Для добавления нового пользователя нажмите кнопку **Создать учетную запись** в верхнем правом углу. Откроется боковая панель создания.

::steps{level="4"}
#### Заполнение данных
Заполните обязательные поля формы:

::field-group
  :::field{name="Логин" type="text" required="true"}
  Уникальный идентификатор пользователя в системе. Минимальная длина — 3 символа.
  :::
  :::field{name="Пароль" type="password" required="true"}
  Первичный пароль для входа.
  :::
  :::field{name="Роль" type="select" required="true"}
  Уровень доступа пользователя (например, *Student* или *Teacher*).
  :::
::

#### Требования к безопасности пароля
Для обеспечения безопасности системы пароль должен соответствовать следующим критериям:
* Минимум **8 символов**.
* Содержать буквы **разного регистра** (A-z).
* Содержать минимум одну **цифру** (0-9).
* Содержать минимум один **спецсимвол** (!@#$%).

#### Сохранение
Нажмите кнопку **Сохранить**. Система автоматически проверит уникальность логина. Если логин занят, вы увидите сообщение об ошибке.
::

## Редактирование и Сброс пароля

Чтобы изменить данные пользователя, нажмите кнопку редактирования (иконка карандаша) в строке таблицы.

### Смена роли
Вы можете изменить роль пользователя в любой момент. Права доступа обновятся немедленно, однако для применения новых политик пользователю может потребоваться перезайти в систему.

### Сброс пароля
Если пользователь забыл пароль, администратор может задать новый вручную.

::note{icon="i-lucide-key"}
В окне редактирования поле **Пароль** по умолчанию пустое.
* Оставьте его пустым, чтобы **сохранить текущий пароль**.
* Введите новый пароль, чтобы **сбросить** старый.
::

## Статусы пользователей

Статус определяет возможность входа в систему.

| Статус | Описание |
| :--- | :--- |
| :badge{label="Активный" color="primary" variant="subtle"} | Пользователь имеет полный доступ к системе согласно своей роли. |
| :badge{label="Временно заблокирован" color="warning" variant="subtle"} | Доступ временно ограничен (например, после серии неудачных попыток входа). Обычно снимается автоматически или вручную. |
| :badge{label="Заблокирован" color="error" variant="subtle"} | Полный запрет доступа. Используется для отчисленных студентов или уволенных сотрудников. |

## Удаление пользователей

Система поддерживает "мягкое удаление" (Soft Delete).

1.  Нажмите кнопку удаления (иконка корзины) в строке пользователя.
2.  Подтвердите действие в модальном окне.

::warning{icon="i-lucide-triangle-alert"}
Удаленная учетная запись помечается как `deleted` в базе данных и скрывается из списка, но физически данные сохраняются для истории и целостности связей.
::
