# Конструктор Курсов

**Конструктор** — это основной инструмент для структурирования образовательного контента. Он позволяет создавать уроки, выстраивать их последовательность и наполнять интерактивными блоками.

## Обзор интерфейса

Рабочее пространство разделено на три функциональные зоны:

::card-group
  ::card{title="1. Структура (Слева)" icon="i-lucide-list-tree"}
  Навигационная панель с иерархией уроков. Используется для создания, выбора и сортировки глав курса.
  ::
  ::card{title="2. Холст (Центр)" icon="i-lucide-layout"}
  Область редактирования. Здесь отображаются и настраиваются контентные блоки (текст, код, видео).
  ::
  ::card{title="3. Инспектор (Справа)" icon="i-lucide-settings-2"}
  Контекстная панель свойств. Отображает настройки для выбранного в данный момент объекта (Урока или Блока).
  ::
::

## Управление уроками

Структура курса состоит из **Уроков**. Урок служит контейнером для блоков контента.

### Создание урока

Для добавления новой структурной единицы:

::steps{level="4"}
#### Добавить
Нажмите кнопку **Добавить урок** в нижней части левой панели.

#### Детали
Заполните форму в модальном окне:

::field-group
  :::field{name="Название" type="text" required="true"}
  Имя урока, которое будет видно студентам.
  :::
  :::field{name="Цели обучения" type="textarea"}
  Опциональное описание ключевых навыков или тем, которые затрагивает урок.
  :::
::

#### Статус черновика
Опция **Сохранить как черновик** включена по умолчанию. Это гарантирует, что урок будет скрыт сразу после создания.

#### Сохранить
Нажмите **Сохранить**. Урок появится в списке структуры.
::

### Система черновиков

Видимость материалов контролируется в секции **Статус** на панели Инспектора.

::alert{icon="i-lucide-eye-off" color="warning"}
**Черновики скрыты от студентов.**
Уроки, отмеченные как черновик, исключаются из учебного плана студента, даже если сам курс опубликован.
::

Состояние урока отображается иконками в боковой панели:
* **Черновик:** :icon{name="i-lucide-file-dashed" class="text-gray-400"} — Контент скрыт.
* **Опубликован:** :icon{name="i-lucide-file-check" class="text-primary-500"} — Контент доступен.

### Сортировка

Порядок уроков может быть изменен в любой момент.

1.  Найдите урок в панели **Структура курса**.
2.  Нажмите и удерживайте ручку захвата (:icon{name="i-lucide-grip-vertical"}) слева от названия урока.
3.  Перетащите элемент на нужную позицию.

Изменения сохраняются автоматически и применяются немедленно.

## Холст и Блоки контента

**Холст** — это центральная рабочая область, где формируется содержание урока. Athena LMS использует упрощенную систему блоков для организации контента.

### Добавление блоков

Для добавления контента в урок:

1.  Нажмите кнопку **Добавить блок** в нижней части холста.
2.  Выберите тип блока:

| Тип | Описание |
| :--- | :--- |
| **Текст** | Универсальный блок для теории. Поддерживает форматирование, изображения, видео и таблицы. |
| **Код** | Интерактивная среда программирования с подсветкой синтаксиса и возможностью автопроверки. |

### Управление блоками

* **Выделение:** Нажмите на блок, чтобы выбрать его. Активный блок подсвечивается синей рамкой.
* **Свойства:** При выборе блока панель **Инспектора** справа обновляется, отображая соответствующие настройки (например, параметры запуска кода).
* **Сортировка:** Используйте ручку захвата (:icon{name="i-lucide-grip-vertical"}) слева от блока для перемещения его вверх или вниз.
* **Удаление:** Чтобы удалить блок, выберите его и нажмите кнопку **Удалить** в нижней части панели Инспектора.

## Текстовый редактор

Блок **Текст** представляет собой мощный редактор на базе Tiptap. Он предназначен для работы не только с текстом, но и со всеми видами медиа-контента.

### Форматирование
Выделите любой фрагмент текста, чтобы открыть **Всплывающее меню** (Bubble Menu) для быстрых действий:
* **Стиль:** Жирный, Курсив, Подчеркнутый, Код в строке.
* **Ссылки:** Вставка гиперссылок.
* **Код:** Преобразование выделения во фрагмент кода.

### Слэш-команды и Медиа
Вы можете вставлять медиафайлы и форматировать контент, не убирая рук с клавиатуры. Напечатайте `/` (слэш) в начале новой строки, чтобы открыть командное меню.

**Доступные команды:**
* **Структура:** Заголовки (H1, H2, H3), Параграф.
* **Списки:** Маркированный список, Нумерованный список.
* **Медиа:**
    * **Изображение:** Вставляет виджет загрузки. Поддерживает Drag-and-Drop.
    * **Видео:** Вставляет виджет загрузки видеофайла.
* **Вставка:** Цитата, Блок кода (статичный), Разделитель.

::note{icon="i-lucide-image"}
**Работа с медиа:**
Изображения и видео встраиваются непосредственно в поток текста. Вы можете взаимодействовать с ними (просматривать, удалять) прямо в рабочей области редактора.
::

## Блок Кода и Исполнение

**Блок кода** превращает обычный урок в интерактивное задание по программированию. Он включает в себя встроенный редактор кода, компилятор/интерпретатор и систему автоматической проверки решений.

### Структура блока

Каждый блок кода в рабочей области состоит из трех частей:
1.  **Текст задания:** Область с форматированием над редактором, где описывается задача.
2.  **Редактор кода:** Интерактивная область, где студенты пишут свое решение.
3.  **Консоль:** Отображает результат выполнения, ошибки или статус проверки.

### Настройки Инспектора

Чтобы настроить параметры выполнения и проверки кода, выберите блок и перейдите к панели **Инспектора**.

#### Среда выполнения
* **Язык:** Выберите язык программирования (например, Python, SQL).
* **Время (мс):** Максимальное время выполнения кода. По умолчанию 1000 мс.
* **Память (МБ):** Максимальный объем потребляемой оперативной памяти. По умолчанию 128 МБ.

::alert{icon="i-lucide-triangle-alert" color="warning"}
**Лимиты ресурсов:**
Если код студента превысит эти ограничения, выполнение будет прервано с ошибкой `Time Limit Exceeded` (TLE) или `Memory Limit Exceeded` (MLE).
::

### Логика проверки

Athena LMS поддерживает два режима проверки решений:

::card-group
  ::card{title="Сравнение вывода (I/O)" icon="i-lucide-arrow-left-right"}
  **Подходит для: Алгоритмов, олимпиадных задач.**
  Система подает данные на **Вход (Stdin)** программы и сравнивает **Выход (Stdout)** с эталонной строкой.
  ::
  ::card{title="Unit-тесты" icon="i-lucide-flask-conical"}
  **Подходит для: Функций, классов, бизнес-логики.**
  Система запускает код студента, а затем выполняет скрытый скрипт с проверками (assertions) для тестирования внутренних переменных или возвращаемых значений.
  ::
::

#### Режим 1: Сравнение вывода (I/O)
В Инспекторе заполните поля:
* **Входные данные (STDIN):** Данные, которые будут переданы программе.
* **Ожидаемый вывод (STDOUT):** Точная строка, которую программа должна вывести для успешного прохождения.

#### Режим 2: Unit-тесты
При выборе этого режима редактор кода на холсте расширяется и показывает дополнительные вкладки:

1.  **Решение:** Начальный код для студента.
2.  **Тесты (Скрыто):** Напишите здесь проверки (например, `assert add(2, 2) == 4`). Этот код выполняется *после* кода студента.
3.  **Начальные скрипты (Скрыто):** Код, который выполняется *перед* кодом студента (например, инициализация базы данных или импорт библиотек).

::note{icon="i-lucide-eye-off"}
**Видимость:**
Вкладки "Тесты" и "Начальные скрипты" видны только преподавателю в Конструкторе. Студенты видят только вкладку "Решение".
::

## Инспектор и Настройки

Панель **Инспектора** справа является контекстно-зависимой. Она динамически изменяет свое содержимое в зависимости от того, что выбрано на Холсте или в Боковой панели.

### Переключение контекста
* **Настройки Урока:** Нажмите на название урока в боковой панели, чтобы увидеть общие свойства (Название, Цели, Статус черновика).
* **Настройки Блока:** Нажмите на конкретный блок на холсте, чтобы настроить его параметры (например, Лимиты кода, URL видео).

### Действие для прохождения (Required Action)

Для каждого блока контента вы должны определить условие, при котором он считается "пройденным". Эта настройка управляет прогрессом студента.

Находится в секции **Основные настройки** Инспектора:

| Тип действия | Описание |
| :--- | :--- |
| **Просмотр (View)** | Студенту достаточно просмотреть контент. По умолчанию для Текста/Изображений. |
| **Отправка (Submit)** | Студент должен выполнить действие с успешным результатом. Обязательно для **Блоков кода** (тесты должны пройти). |
| **Взаимодействие (Interact)** | Требует активного участия (например, запуск видео). |
| **Пропуск (Pass)** | Действие не требуется; информационный блок. |

::note{icon="i-lucide-check-circle"}
**Рекомендация:**
Всегда устанавливайте для **Блоков кода** значение `Submit` (Отправка). Если оставить `View`, студент сможет пропустить задачу по программированию, не написав ни строчки кода.
::

## Режим предпросмотра

Перед публикацией изменений важно проверить, как контент будет выглядеть для конечного пользователя.

### Включение предпросмотра
Нажмите кнопку **Предпросмотр** (:icon{name="i-lucide-eye"}) в верхней навигационной панели.

### Что происходит в этом режиме?
1.  **Интерфейс "Только для чтения":** Все инструменты редактирования, панели инструментов и ручки перетаскивания скрываются.
2.  **Опыт студента:** Вы можете взаимодействовать с блоками кода, запускать тесты и просматривать медиа точно так же, как это будет делать студент.
3.  **Чистый макет:** Панель Инспектора скрывается, чтобы максимально увеличить область чтения.

Чтобы вернуться к редактированию, нажмите кнопку **Редактировать** (:icon{name="i-lucide-pencil"}) в заголовке.
